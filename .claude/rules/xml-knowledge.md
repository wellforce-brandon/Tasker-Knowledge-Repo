# XML Knowledge Rules

When editing files in `knowledge/xml/`:

- Always include XML snippets demonstrating the concept.
- Document action code numbers alongside their human-readable names.
- Note which Tasker version introduced or changed XML format elements.
- Use properly indented XML in fenced code blocks with `xml` language tag.
- When documenting attributes, include the attribute name, type, and valid values.
