# Plugin Knowledge Rules

When editing files in `knowledge/plugins/`:

- Document the plugin version where behaviors were tested.
- Note any known conflicts with other plugins or Tasker versions.
- Include bundle extra names and intent action strings for plugin actions.
- Document both the GUI configuration and the equivalent intent/bundle setup.
- Note if a plugin feature requires a paid/pro version.
